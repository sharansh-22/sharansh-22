My work focuses on financial machine learning, quantitative risk modeling, and AI systems. Rather than optimizing solely for benchmark performance, I prioritize understanding **why models succeed, why they fail, and what evidence justifies architectural decisions.**

# Selected Projects

## Cascade Risk Intelligence System (CRIS)

*A modular financial data and feature engineering platform for quantitative machine learning.*

CRIS is a layered architecture for transforming heterogeneous financial data into validated feature spaces for downstream risk models. The system emphasizes reproducibility, causal correctness, and strict separation between data engineering, feature generation, and predictive modeling.

**Highlights**

- Dagster-orchestrated ETL pipelines
- Multi-source financial data ingestion
- Schema validation and data quality reporting
- Polars-based feature engineering
- Zero look-ahead bias
- Configuration-driven signal extraction

**Technologies**

`Python` Рђб `Dagster` Рђб `Polars` Рђб `Pandas`

## Consumer Credit Risk Platform

*A research-driven platform for consumer credit default prediction.*

Built on 1.34 million LendingClub loans, the project investigates both predictive performance and the limitations of borrower-centric credit models. Beyond model development, the platform studies **Stealth Defaulters**Рђћborrowers who systematically evade prediction despite extensive feature engineering.

**Highlights**

## Retrieval-Augmented Generation Laboratory (RAGL)

*A scientific framework for designing and validating Retrieval-Augmented Generation architectures.*

Instead of continually adding components to RAG systems, RAGL evaluates architectural changes through controlled experiments. Every modification is benchmarked against immutable reference architectures before adoption.

**Highlights**

- Controlled experimentation
- Immutable baselines
- Benchmark automation
- Engineering review workflow
- Architecture validation

**Technologies**

`Python` Рђб `FAISS` Рђб `Ollama`
